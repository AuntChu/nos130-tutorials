Section 1 of 5

Local vs Microsoft Accounts

Two sign-in options. Very different implications for sync, cloud access, and management.

Local Account

Username and password stored on this machine only. No sync. No automatic OneDrive. No email recovery. Works fully offline. Best for shared, managed, or isolated machines.

Microsoft Account

Microsoft email as sign-in. Settings sync across devices. OneDrive connects automatically. Password recoverable by email. Best for personal devices and remote workers.

Section 2 of 5

Standard User vs Administrator + UAC

Two roles. One elevation system that protects both.

Standard User STD

Run installed apps, change personal settings, manage own files. Cannot install drivers, modify system settings, or change other users' files.

Administrator ADM

Everything Standard User can do, plus: install software and drivers, modify system settings, create and delete any user account, access all files.

User Account Control (UAC)

When any action requires elevation, UAC intervenes. Standard Users must supply Administrator credentials. Administrators see a Yes/No confirmation prompt — they are not exempt from UAC.

Section 4 of 5

Windows Hello

Device-bound credentials that are more secure than passwords — once you understand why.

The security principle

All Hello credentials are stored on the device in the TPM chip. Nothing is sent to Microsoft. A PIN or biometric only works on the specific machine where it was enrolled — it cannot be used remotely, phished, or replayed on another device.

The three methods

PIN — Always available. Numbers only or letters and symbols. Device-specific. Set up in Settings → Accounts → Sign-in options → PIN.

Facial Recognition — Requires IR camera. Uses depth mapping, not photo matching. Cannot be fooled by a printed photo.

Fingerprint — Requires fingerprint reader. Enrol at least two fingers. Falls back to PIN automatically.

Section 5 of 5

Hello for Business, LAPS & Family Safety

Enterprise identity tools and family account management.

Windows Hello for Business

Enterprise version of Hello. Credential tied to a certificate issued by Entra ID or Active Directory. Managed by Intune or Group Policy. Cannot be phished — no password to steal.

Windows LAPS

Automatically generates, rotates, and stores a unique local administrator password per device. Prevents lateral movement attacks where one leaked admin password unlocks every machine.

Microsoft Family Safety

Cloud-based family management tied to Microsoft accounts. Controls work across devices — Windows, Xbox, Android. Requires child to be signed in with a Microsoft account. Local accounts are invisible to Family Safety.

Five things to remember

▸  Microsoft accounts sync settings and connect OneDrive automatically. Local accounts stay on the device.

▸  Standard Users cannot install software or change system settings. UAC prompts both Standard Users and Administrators.

▸  A Hello PIN only works on the device it was set on. That is what makes it more secure than a password.

▸  Windows LAPS prevents lateral movement by giving every managed device a unique local admin password.

▸  Family Safety requires the child's Microsoft account. Local accounts are invisible to it.
