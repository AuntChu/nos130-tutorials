03The Stolen Laptop

A laptop with your Unit 4 NTFS permissions gets stolen. Run the attack, then flip one switch and run it again.

BitLocker: OFF (drive is plaintext, protected only by NTFS)
1

Pull the drive

Ten minutes with a screwdriver. The sign-in screen never appears.

2

Mount it elsewhere

Another PC, or boot the laptop itself from a USB stick running a different OS.

3

Read the disk

The other OS reads raw NTFS structures directly.

FILES READABLE. The HR folder, the payroll spreadsheet, everything. Your NTFS permissions are still written on the disk, politely, but they are rules that Windows enforces, and the thief's OS simply does not ask. Permissions died with the operating system.
CIPHERTEXT. The thief mounts the drive and finds scrambled noise. The decryption key never lived on the disk; it stayed behind in the TPM. Without the key or the 48-digit recovery code, the data does not exist in any usable sense.
