Choosing the Right View

Right-click an empty area in File Explorer and choose View, or use the View button in the ribbon. Here is when to use each option:

  • Details — Best for IT work. Shows name, date, type, and size in sortable columns.
  • Large Icons — Best for photo and media folders where thumbnails matter.
  • List — Compact. Useful when you need to scan hundreds of files quickly.
  • Content — Shows a text preview snippet next to each file. Useful for document-heavy folders.
TIP

In Details view, click any column header to sort by that field. Click again to reverse the sort order. Shift-click a second column to add a secondary sort.

SECTION 2

Quick Access and Libraries

Two different tools for organizing how you navigate to files.

Quick Access
  • Appears at the top of the navigation pane
  • Auto-populates with recent and frequent folders
  • Pin any folder: right-click → Pin to Quick Access
  • Unpin: right-click in nav pane → Unpin from Quick Access
  • Best for: personal shortcuts to your most-used locations
Libraries
  • Hidden by default — enable via View → Navigation pane
  • Virtual containers aggregating multiple physical folders
  • Add a folder: right-click → Include in library
  • Default libraries: Documents, Music, Pictures, Videos
  • Best for: viewing files from multiple locations in one place

When to Use Each One

  • Quick Access is personal and immediate — think of it as your personal shortcuts panel. Use it for folders you navigate to multiple times a day.
  • Libraries are more powerful when content lives in multiple places. A Library called "Project X" could show files from a local folder, a network share, and an external drive all at once — without moving anything.
HOW TO ENABLE LIBRARIES

In File Explorer, click View in the ribbon → Show → Navigation pane → check Show libraries. Libraries will then appear in the left navigation pane.

SECTION 3

NTFS Permissions

Controlling who can read, write, and modify files at the file system level.

The Six Standard Permission Levels

Full Control
Read, write, delete, change permissions, take ownership
Modify
Read and write files; delete files and subfolders
Read & Execute
View contents and run programs; no changes permitted
Read
Open and view files only; cannot modify anything
Write
Add files and subfolders; cannot read existing content
Special
Fine-grained combination of individual permission flags

How to Set Permissions

  1. Right-click the file or folder → Properties
  2. Click the Security tab
  3. Click Edit to open the permission dialog
  4. Click Add to search for a user or group, or select an existing entry
  5. Check Allow or Deny boxes for each permission level
  6. Click Apply, then OK
ALLOW vs DENY

Deny always overrides Allow. If a user is in two groups and one has Allow Read while the other has Deny Read, the user cannot read the file. Use Deny carefully — in most cases, you manage access by adding or removing Allow permissions instead.

Inheritance

By default, subfolders and files inherit permissions from their parent folder. This means if you grant Read to a group on a top-level folder, everything inside it gets Read automatically.

To give a subfolder its own independent rules, open its Advanced Security Settings and click Disable Inheritance. Choose to convert the inherited permissions to explicit ones, then modify as needed.

Inheritance ON (default)

Permissions flow from parent to all children automatically. Changes at the parent level propagate down. Easiest to maintain for large folder structures.

Inheritance OFF (explicit)

Folder has its own independent permission set. Parent changes do not affect it. Required when stricter or different rules are needed for a specific subfolder.

Quick Check: Allow or Deny?

?
A user is a member of "Managers" (Allow Modify) and "Auditors" (Deny Write). Can the user modify files in a shared folder?

SECTION 4

File Compression

Two different compression tools built into Windows — and when to use each one.

ZIP Compression
  • Right-click → Compress to ZIP file
  • Creates a portable .zip archive
  • Cross-platform: works on Mac, Linux, all Windows versions
  • Extract with double-click or right-click → Extract All
  • No encryption by default (use 7-Zip for passwords)
  • Best for: sharing or emailing collections of files
NTFS Compression
  • Right-click → Properties → Advanced → Compress contents
  • Transparent — files open normally, no manual unzip needed
  • NTFS volumes only (not FAT32, exFAT)
  • Compressed items appear in blue in File Explorer
  • Not recommended for JPG, MP4, or other pre-compressed files
  • Best for: archiving large text-heavy files on the same drive

Which Would You Choose?

1
You need to email 30 Word documents to a client on a Mac. Which compression method should you use?
ZIP — creates a portable archive that works cross-platform
NTFS Compression — transparent and doesn't require extracting
2
You have 20 GB of old server log files (.txt) on an NTFS drive that you rarely access but need to keep locally. Which method is better?
ZIP — more portable and widely compatible
NTFS Compression — transparent, stays on the same NTFS volume, no manual extraction needed
SECTION 5

OneDrive Integration

Cloud storage, sync, and the Files On-Demand feature.

The Three File States

☁️ Cloud Only
  • Visible in File Explorer
  • Not stored locally
  • Downloads when opened
  • Shows cloud icon in Status column
✅ Available Locally
  • Downloaded to local drive
  • Available offline
  • Uses disk space
  • Shows green checkmark
🔒 Always on Device
  • Permanently local copy
  • Never auto-removed by OneDrive
  • Uses disk space permanently
  • Shows filled green circle

Managing File States

  • Right-click a cloud-only file → Always keep on this device to download it permanently.
  • Right-click a local file → Free up space to return it to cloud-only and reclaim disk space.
  • To see the Status column in File Explorer: right-click any column header in Details view → check Status.
IT SUPPORT TIP

When a user says they are running out of disk space and OneDrive is enabled, check the Status column. If many files show green checkmarks (local), use Free up space on older files to move them back to cloud-only. This is often faster than uninstalling programs.

Command-Line File Operations

Managing files from cmd.exe and PowerShell — side by side.

Command Prompt (cmd.exe)

cmd.exe — Command Prompt
C:\Users\Student> dir <-- list contents
C:\Users\Student> cd Documents <-- change directory
C:\Users\Student\Documents> mkdir Reports <-- new folder
C:\Users\Student\Documents> copy file.txt D:\ <-- copy file
C:\Users\Student\Documents> move file.txt .\sub\ <-- move file
C:\Users\Student\Documents> ren old.txt new.txt <-- rename
C:\Users\Student\Documents> del file.txt <-- delete

PowerShell

Windows PowerShell
PS C:\Users\Student> Get-ChildItem <-- list contents
PS C:\Users\Student> Set-Location .\Documents <-- change directory
PS C:\> New-Item -ItemType Directory -Name Reports <-- new folder
PS C:\> Copy-Item file.txt D:\ <-- copy file
PS C:\> Move-Item file.txt .\sub\ <-- move file
PS C:\> Rename-Item old.txt -NewName new.txt <-- rename
PS C:\> Remove-Item file.txt <-- delete
PS C:\> Get-ChildItem -Recurse <-- list all nested files
ALIASES IN POWERSHELL

PowerShell includes aliases so that classic cmd commands work too: dir, cd, copy, move, del, mkdir. These are shortcuts to the underlying cmdlets. You can use either style in PowerShell.
